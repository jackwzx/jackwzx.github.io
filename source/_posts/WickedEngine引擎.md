**WickedEngine** 是一个开源的、基于 C++ 编写的高性能游戏引擎，以其**先进的渲染技术**和**极高的代码质量**在图形编程和引擎开发社区中闻名。

它最初由匈牙利开发者 **Turánszki János** 作为一个业余项目开发，但如今已经发展成为一个功能相当完备，且在图形技术上紧跟业界前沿（如支持光线追踪、Mesh Shaders 等）的引擎。

以下是关于 WickedEngine 的详细介绍：

### 1. 核心定位
*   **类型**：开源游戏引擎 (MIT 协议)。
*   **语言**：核心为 C++，脚本支持 Lua。
*   **平台**：主要支持 Windows (DirectX 11/12)，也支持 Linux (Vulkan) 和 UWP (Xbox)。
*   **目标人群**：图形程序员、引擎开发学习者、技术演示开发者，以及想要深入理解现代渲染管线的人。

### 2. 核心特性 (为什么它很厉害？)

WickedEngine 最大的卖点在于其**图形渲染能力**，它往往能以极快的速度跟进最新的图形 API 特性。

#### A. 现代渲染管线
*   **API 支持**：支持 DirectX 11, **DirectX 12**, **Vulkan**。
*   **光线追踪 (Hardware Ray Tracing)**：完全支持 DXR (DirectX Raytracing)，包括光追阴影、反射、环境光遮蔽 (RTAO) 和全局光照。
*   **全局光照 (GI)**：除了硬件光追 GI，它还实现了基于体素的 GI (Voxel GI) 和屏幕空间技术，能在非光追硬件上实现出色的光照效果。
*   **后处理**：内置了包括 Bloom, Motion Blur, DOF (景深), SSAO/HBAO, SSR (屏幕空间反射), TAA/FXAA 等全套现代后处理特效。
*   **下一代特性**：支持 **Mesh Shaders**（网格着色器）和 **Variable Rate Shading**（可变着色率，VRS），这是很多商业引擎都在逐步推进的技术。

#### B. 高性能架构
*   **多线程 (Job System)**：引擎底层设计了高效的任务系统，渲染、物理、逻辑更新都能充分利用多核 CPU。
*   **无依赖地狱**：WickedEngine 的一个显著特点是**外部依赖极少**。它尽量自己造轮子或使用轻量级库，这使得编译和集成它非常容易（通常只需要几分钟就能编译完）。
*   **实体组件系统 (ECS)**：虽然不是严格的纯数据 ECS，但采用了类似的组件化设计，内存布局紧凑，缓存友好。

#### C. 功能模块
*   **物理**：集成了 **Bullet Physics** 引擎。
*   **脚本**：深度集成了 **Lua**，几乎所有引擎功能都可以通过 Lua 控制，支持热重载。
*   **编辑器**：自带一个全功能的编辑器，支持场景搭建、材质编辑、后处理调节、粒子编辑等。虽然UI不如 Unreal 精美，但功能非常实用。
*   **声音**：自研的音频系统。

### 3. 为什么你要关注 WickedEngine？

如果你是以下几类人，WickedEngine 是一个宝藏：

1.  **图形编程学习者**：
    *   WickedEngine 的代码结构非常清晰，是学习如何封装 DX12/Vulkan、如何实现现代延迟渲染管线（Deferred Rendering）、如何写 Compute Shader 的绝佳教材。
    *   相比于 Unreal 庞大复杂的源码，WickedEngine 更容易“读懂”并魔改。

2.  **C++ 进阶开发者**：
    *   想看看如何用现代 C++ 写一个高性能、低依赖的系统？看它的源码。

3.  **独立技术演示 (Demo) 制作**：
    *   如果你不需要复杂的 Gameplay 逻辑，只需要极其炸裂的画面（比如建筑漫游、画质测试），WickedEngine 极其轻量且效果拔群。

### 4. 与其他引擎的对比

| 特性 | WickedEngine | Unreal Engine (UE) | Godot |
| :--- | :--- | :--- | :--- |
| **渲染上限** | **极高** (接近 3A 标准) | **顶尖** (行业标准) | 中等偏高 (4.0 有提升) |
| **易用性** | 低 (程序员导向) | 中 (有蓝图，但系统复杂) | 高 (上手简单) |
| **生态/资源** | 极少 (几乎没有商店) | 极大 (商城、教程无数) | 较大 |
| **源码复杂度** | **低/中** (适合阅读学习) | 极高 (数百万行代码) | 中 |
| **主要用途** | 学习、图形Demo、自研 | 3A游戏、影视、模拟 | 独立游戏 |

### 5. 如何开始？

1.  **获取源码**：
    GitHub 搜索 `WickedEngine` (作者是 `turanszkij`)。
2.  **编译**：
    下载后，只需安装 Visual Studio 和 Windows SDK，打开解决方案文件，通常**一键编译**即可运行，几乎不需要配置环境。
3.  **运行 Editor**：
    编译生成的 `Editor.exe` 可以让你直接拖入 glTF 模型进行渲染测试。

### 总结
**WickedEngine 是目前开源界图形技术最激进、代码最干净的 C++ 引擎之一。** 它可能不是用来做商业手游的最佳选择（工具链不如 Unity/UE 成熟），但绝对是图形程序员的“圣经”级参考项目。如果你想从底层理解现代渲染技术，而不是仅仅停留在调用 API，WickedEngine 值得深入研究。
